# Docusaurus Docs Navigation Restructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `development`、`operations`、`software` 文档统一迁移为 `教程` 与 `踩坑` 两套目录和 navbar 下拉入口，并保持 `nebula` 完全不变。

**Architecture:** `docs/nebula` 与 `sidebars/nebula.ts` 不改动。迁移源为当前仓库真实目录 `docs/development`、`docs/operations`、`docs/software`；迁移目标为 `docs/tutorials` 与 `docs/troubleshooting`，各场景目录通过 `_category_.json` 生成场景索引，navbar 使用 Docusaurus dropdown 指向自动生成的文档 URL。

## File Structure

- `docs/tutorials/**`: 教程类文档，按 `linux/windows/macos/docker/k8s/ci/java/database/network/other` 场景归档。
- `docs/troubleshooting/**`: 踩坑类文档，按同样场景归档。
- `docs/**/assets/**`: 有图片或附件的文档必须使用 `文档目录/index.md + assets/` 结构。
- `sidebars/index.ts`: 注册 `nebulaSidebar`，并按场景拆分 `tutorials*Sidebar` 与 `troubleshooting*Sidebar`。每个场景 sidebar 使用 `autogenerated` 指向对应子目录，例如 `troubleshooting/docker`，避免进入 Docker 时显示整棵踩坑树。
- `docusaurus.config.ts`: navbar 保留 Nebula，移除旧开发/软件/运维入口，新增 `教程` 与 `踩坑` dropdown。

## Migration Rules

- [ ] 不修改 `docs/nebula/**` 和 `sidebars/nebula.ts`。
- [ ] 旧 `development/operations/software` 目录只作为迁移源，迁移后不再参与构建。
- [ ] 有资源的单文件文档从 `xxx.md` 改为 `xxx/index.md`，资源放入 `xxx/assets/`。

- [ ] 文档间引用统一改为 Docusaurus 生成路径，不带 `.md`，不指向 `index`。
- [ ] 迁移文档删除 front matter 中的 `slug` 与 `id`，URL 交给 Docusaurus 根据目录生成。
