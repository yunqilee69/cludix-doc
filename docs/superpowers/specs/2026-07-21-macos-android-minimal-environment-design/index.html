macOS Android 最小开发环境教程设计

目标​

新增一篇 macOS 教程，指导读者仅安装 Android 真机开发与 APK 构建所需的最小 Android 环境，不将 Android Studio、Android Emulator 或 APK 发包流程纳入范围。

读者与边界​

  • 读者已使用 macOS，并希望通过命令行完成 Android SDK 的安装与配置。
  • 读者需要连接真机、使用 adb、下载项目要求的 Android 平台，并构建 Android 项目。
  • JDK 不在本文重复安装；本文仅要求已安装 JDK 17 或 21，并链接至现有 SDKMAN! 安装与 Java 多版本管理。
  • Android Studio 与 Android Emulator 明确标为非必需组件，不提供安装步骤。
  • APK 发包、签名与发布流程由后续独立教程覆盖。

文档归属​

  • 文档类型：教程。

  • 主目录：docs/tutorials/macos/。

  • 新文档：docs/tutorials/macos/macOS配置Android开发环境.md。

  • Front matter：

    ---
    title: macOS 配置 Android 最小开发环境
    date: 2026-07-21 00:00
    tags: [macos, java, installation, configuration]
    ---

内容结构​

  1. 说明最小环境的目标、预计磁盘占用（约 1.5–2 GB）与非必需组件。
  2. 列出五项必要组件及职责：JDK、Command-line Tools、Platform-Tools、Build-Tools、SDK Platform。
  3. 链接到现有 SDKMAN! 教程，并使用 java -version 验证 JDK。
  4. 从 Android 官方下载页获取 Command-line Tools；解压为 $HOME/Library/Android/sdk/cmdline-tools/latest，使 sdkmanager 处于预期目录。
  5. 在 ~/.zshrc 配置 ANDROID_HOME、ANDROID_SDK_ROOT 与 SDK 工具路径，并通过 source ~/.zshrc 使配置立即生效。
  6. 使用 sdkmanager 接受许可证并安装以下组件：
    • platform-tools
    • build-tools;34.0.0
    • platforms;android-34
  7. 说明 API 34 与 Build-Tools 版本只是示例，实际版本应与项目的 compileSdk / 构建要求保持一致。
  8. 通过 sdkmanager --list、adb version、adb devices 完成验证；说明开启开发者选项、USB 调试和设备授权的真机连接步骤。
  9. 给出三项高频排查：sdkmanager 不存在、设备未被 adb 识别、Android SDK 许可证未接受。

导航改动​

  • 更新手动侧边栏配置：新增 macOS 教程侧边栏并包含目录索引、SDKMAN! 文档和新 Android 环境教程；在根侧边栏注册该配置。

验收标准​

  • 新教程包含有效的 date 和白名单内的 tags。
  • 所有文档互链使用相对路径且不包含 .md 后缀。
  • 教程内容明确区分必需组件与非必需组件。
  • 命令按 macOS 默认 Zsh 环境编写，且 SDK 目录结构能被 sdkmanager 识别。
  • npm run check-doc-tags、npm run typecheck 与 npm run build 通过。
