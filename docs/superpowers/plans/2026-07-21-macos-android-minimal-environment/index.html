# macOS 配置 Android 最小开发环境

本文只安装 Android 真机开发与 APK 构建需要的最小组件，预计占用约 1.5–2 GB 磁盘空间。

本文不安装 Android Studio 或 Android Emulator；它们是可选工具，不影响命令行构建和真机调试。APK 签名、发包与发布流程会在独立文档中说明。

## 1. 准备 JDK

Android 构建需要 JDK。推荐使用 JDK 17 或 JDK 21。

如果尚未安装 JDK，请先阅读 [SDKMAN! 在 macOS 上的安装与 Java 多版本管理](./sdkman-java/index)。

安装后验证：

```bash
java -version

2. 安装 Android SDK Command-line Tools​

从 Android Developers Command-line tools 下载 macOS 版 commandlinetools-mac-*.zip，然后执行：

export ANDROID_SDK_ROOT="$HOME/Library/Android/sdk"
mkdir -p "$ANDROID_SDK_ROOT/cmdline-tools"
unzip ~/Downloads/commandlinetools-mac-*.zip -d "$ANDROID_SDK_ROOT/cmdline-tools"
mv "$ANDROID_SDK_ROOT/cmdline-tools/cmdline-tools" \
  "$ANDROID_SDK_ROOT/cmdline-tools/latest"

## 3. 配置环境变量

将以下内容追加到 `~/.zshrc`：

```bash
export ANDROID_HOME="$HOME/Library/Android/sdk"
export ANDROID_SDK_ROOT="$ANDROID_HOME"
export PATH="$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools:$PATH"

重新加载配置：

source ~/.zshrc
sdkmanager --version

4. 安装必要 SDK 组件​

sdkmanager --licenses
sdkmanager \
  "platform-tools" \
  "build-tools;34.0.0" \
  "platforms;android-34"

这里的 API 34 和 Build-Tools 34.0.0 仅为示例。请查看 Android 项目的 compileSdk 与构建配置，改用项目要求的 API 和 Build-Tools 版本。


- [ ] **Step 5: 写入验证、真机连接与故障排查章节**

Add verification commands and three troubleshooting entries:

```md
## 5. 验证安装并连接真机

```bash
sdkmanager --list
adb version
adb devices

在 Android 手机中依次开启开发者选项和 USB 调试；通过 USB 线连接 Mac 后，在手机弹出的 RSA 调试授权对话框中选择允许。adb devices 显示设备序列号及 device 状态即表示连接成功。

常见问题​

sdkmanager: command not found​

确认 Command-line Tools 位于 $ANDROID_HOME/cmdline-tools/latest/，并重新执行 source ~/.zshrc。

adb devices 没有设备​

确认 USB 调试已开启、数据线支持传输数据，并在手机上允许 RSA 调试授权；随后执行 adb kill-server && adb start-server 后重试。

构建提示 SDK 许可证未接受​

执行 sdkmanager --licenses，逐项输入 y 接受许可证后重新构建。

# macOS

本目录收录 macOS 上的软件安装、终端配置和开发环境搭建文档。

## 文档清单

| 文档 | 说明 |
| --- | --- |
| [Homebrew 在 macOS 上的安装与国内源配置](./homebrew/index) | 安装 Homebrew 并配置常用国内下载源。 |
| [Oh My Zsh配置](./oh-my-zsh/index) | 配置 Zsh 框架及常用终端初始化项。 |
| [Rust 环境与 Cargo 配置（macOS）](./rust-cargo/index) | 安装 Rust 与配置 Cargo 镜像。 |
| [SDKMAN! 在 macOS 上的安装与 Java 多版本管理](./sdkman-java/index) | 安装与切换 Java、Maven 等 SDK。 |
| [Mac mini 设置常开不休眠，但允许屏幕自动熄灭](./macos-mini-power/index) | 让 Mac mini 持续运行且允许显示器熄灭。 |
| [macOS 配置 Android 最小开发环境](./macOS配置Android开发环境) | 安装命令行 Android SDK，并配置真机开发所需工具。 |

## 使用建议

1. 首次配置开发机时，先安装 [Homebrew](./homebrew/index) 与 [SDKMAN!](./sdkman-java/index)。
2. 配置 Android 真机开发环境时，阅读 [macOS 配置 Android 最小开发环境](./macOS配置Android开发环境)。
3. 需要其他开发工具时，再按具体技术栈查看对应教程。
