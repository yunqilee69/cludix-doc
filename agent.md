# 文档新增规范（供后续 Agent 自动执行）

本文件用于约束后续新增/修改文档时的默认行为，目标是：
- 新增文档后无需再手动改 sidebar
- 目录型文档（`index.md`）在侧边栏可点击进入
- 内链不因 slug 调整而频繁失效

## 1. 文件与命名

- 文档文件名优先使用中文（如：`文件上传报错.md`、`Nginx部署.md`）
- 目录名保持现有结构，不强制改中文
- 目录型文档统一使用 `index.md`

## 2. Front Matter 规则

每篇文档必须包含 front matter：

```md
---
slug: /模块/路径
title: 页面标题
---
```

- `slug` 用于稳定 URL（可英文/拼音风格，保持一致即可）
- `title` 用于页面标题与侧边栏显示（未覆盖 label 时）

## 3. 侧边栏策略（自动生成）

当前项目已使用 `autogenerated` 侧边栏（见 `sidebars/*.ts`），新增文档默认不需要手改 sidebar。

### 3.1 `_category_.json` 何时需要 `link`

- **目录内有 `index.md`**：必须在该目录 `_category_.json` 增加 `link`，使分类标题可点击进入目录文档
- **目录内没有 `index.md`**：不要配置 `link`，分类仅作为展开/收起容器

示例（有 `index.md`）：

```json
{
  "label": "Docker",
  "position": 2,
  "collapsible": true,
  "collapsed": true,
  "link": {
    "type": "doc",
    "id": "operations/docker/index"
  }
}
```

示例（无 `index.md`）：

```json
{
  "label": "网络",
  "position": 4,
  "collapsible": true,
  "collapsed": true
}
```

## 4. 文档内链接规则

- 文档互链统一使用**相对文件路径**（推荐带 `.md`）
- 不再使用 `/docs/...` 绝对路径

示例：

```md
[开发工具](./tools/git/index.md)
[问题解决](./troubleshooting/general/文件上传报错.md)
```

## 5. 新增文档操作清单（必须执行）

1. 新建中文文件名文档，补齐 `slug` + `title`
2. 若为目录页：创建/更新 `index.md`
3. 检查对应目录 `_category_.json`
   - 有 `index.md` -> 配置 `link.type=doc`
   - 无 `index.md` -> 移除/不添加 `link`
4. 文档内跳转改为相对链接
5. 验证：
   - `npm run build`
   - `npm run typecheck`

## 6. 维护约定

- 不随意改已有 `slug`（避免外链失效）
- 目录结构调整时，优先修正相对链接与 `_category_.json`
- 若出现“侧边栏分类可展开但不可点击”，优先检查该目录是否缺少 `link`
